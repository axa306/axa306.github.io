Project Summary

In CSC: 431: Introduction to Software Engineering, as a team, we were tasked with creating and designing a project.

The project is an application that would be an alternative to a bloomberg terminal. It takes the data from filings in the SEC database and converts it into readable data -- by using graphics and charts -- for all investors in the stock market.

An example of how this application would be used is the user would search for a stock by its ticker symbol. Then, the ticker symbol would resolve into a CIK number. With the CIK, it is used to search the SEC database to retrieve XBRL files. The XBRL files are then converted into JSON files. Finally the JSON files are used to diplsay graphics and charts for the user.

For the final presentation in class, the two section I presented and designed were 'design patterns' and the 'class diagram'.

Design Patterns

For this project, the adapter design pattern and the command design patterns are design patterns that would help the overall design of the project when creating the software.

The adapter design pattern would provide an object oriented solution to coverting XBRL files to JSON. The adapter design pattern would be an object that is specifically dedicated to translating XBRL files into JSON. The object would recieve input from the files that were retrieved from teh SEC database, then they would be converted and sent out as JSON to the terminal where they would be used to display data in the form of graphics and charts.

Two of the main feature in the project are 'Monitor insider holding' and to 'Request fundamentals'. With the command design pattern, these two request features would be encapsulated as objects. This would allow the software design to decouple classes and seperate the request -- request fundamentals and view insider holdings -- from the actual object. The main object would be a stock.

Class Diagram Overview

The image below is the class diagram I designed for this project. With this diagram, I used an object oriented design to provide an outline of how the program would be implemented.

The user object would have the ability to search for stock, view insider holding and requst fundamentals. The search object would recieve a ticker symbol as a string and would have the ability to validate it and search for the specified symbol in the datbase.

The database would interact with the adapter object by returning the files that are associated with the CIK number. The adapter class would convert and translate the XBRL files that were retieved and convert them to JSON to then output the files to the display object.

The display object would retrieve the JSON files and display graphics and charts. The stock is the main object where 'request fundamentals' and 'view insider holdings' are decoupled from the stock class.
